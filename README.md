# Flight Booking System - Travel AgileWay

## Project Overview
This project involved **manual testing** of the Flight Booking web application **Travel AgileWay**, focusing on the key pages: **Select Flight**, **Passenger Details**, and **Payment**. The testing covered both **valid and invalid scenarios** to ensure the application handles all cases correctly.

## My Role
- Designed and documented **scenarios and detailed test cases** for multiple workflows  
- Created **test cycles** and executed test cases  
- Logged **defects and wrote comprehensive bug reports** for failed cases  
- Managed all testing activities using **Jira**, **Zephyr**, and **Trello**

## Pages Tested
- **Select Flight** – Flight selection, availability, and validation  
- **Passenger Details** – Input validation and error handling for personal information  
- **Payment** – Card validation, expiry checks, and payment processing  

## Tools Used
- **Jira** – Bug tracking and defect management  
- **Zephyr** – Test case management and execution  
- **Trello** – Test planning and workflow tracking  
- **Manual Testing** – End-to-end functional testing  

## Skills Highlighted
- Test case design for valid and invalid inputs  
- Test cycle planning and execution  
- Defect reporting and management  
- Workflow management with Jira, Zephyr, and Trello
